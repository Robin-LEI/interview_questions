# 事件代理（委托）的原理

利用事件冒泡的机制，将后代元素的事件委托给祖先元素。



# 应用场景

1. 可以解决动态绑定事件的问题，常规的做法是新增元素必须分别绑定一个事件处理函数，会造成大量的内存消耗
2. 大量类似元素需要批量添加事件监听



# e.stopPropagation()

阻止事件传播



# onmouseenter和onmouseover都表示鼠标进入，他们的区别是？

前者不冒泡，后者冒泡。

onmouseleave不冒泡，onmouseout冒泡。



# 使用事件委托时，需要注意，不能委托不冒泡的元素给祖先元素。



# 检查一个元素是否冒泡，e.bubbles是否为true，true表示冒泡，false表示不冒泡
